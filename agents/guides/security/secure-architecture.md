# Guide: Secure Architecture & Zero Trust
<!-- Defense in Depth, Micro-segmentation, Least Privilege -->

---

## 1. Zero Trust Architecture (ZTA)
"Never trust, always verify."

- **The Old Perimeter Model**: A firewall protects the outside; everything inside the network is trusted. (Obsolete, highly vulnerable to lateral movement).
- **Zero Trust**: No network is implicitly trusted.
  - Every user, device, and application must authenticate.
  - Every connection is encrypted (mTLS).
  - Access policies are dynamic, based on context (location, device health, anomalies).

## 2. Principle of Least Privilege (PoLP)
Entities should only have the exact permissions necessary to perform their required tasks, and no more.

- **Databases**: The web application should not connect to the database as the `root` or `postgres` user. Create specific roles for read/write.
- **Cloud IAM**: Microservices must run using dedicated IAM roles (e.g., AWS Task Roles) rather than sharing long-lived broad credentials.
- **Containers**: Run Docker containers as non-root users (`USER 1000:1000` in Dockerfile). Drop unnecessary Linux capabilities.

## 3. Defense in Depth
Implement overlapping layers of security. If one fails, another catches the breach.

1. **Edge**: WAF (Web Application Firewall), DDoS protection, Cloudflare.
2. **Network**: VPCs, private subnets, strict Security Groups / Network Policies.
3. **Application**: Input validation, parameterized queries, rate limiting.
4. **Data**: Field-level encryption, KMS, backups.
5. **Monitoring**: Intrustion Detection Systems (IDS), audit logging, SIEM alerts.

## 4. Secure Development Lifecycle (SDLC)
Shift security to the "Left" (early in the process).

- **SAST**: Static Application Security Testing (e.g., Semgrep, SonarQube) in CI pipelines to catch hardcoded secrets or bad patterns.
- **SCA**: Software Composition Analysis (e.g., Dependabot, Snyk) to block vulnerable open-source libraries.
- **DAST**: Dynamic Application Security Testing to scan running applications for vulnerabilities.

## 5. Supply Chain Security
Modern apps use thousands of dependencies. You must protect the build pipeline.

- **Dependency Pinning**: Lock all dependency versions (e.g., `package-lock.json`, `requirements.txt`).
- **Reproducible Builds**: Given the same source code, the build process should always produce the exact same binary artifact.
- **Provenance**: Sign your Docker images and artifacts (e.g., Sigstore/Cosign) to prove they were generated by your trustworthy CI/CD system and not tampered with.
